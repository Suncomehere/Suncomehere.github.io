Hexo博客搭建

1.Hexo介绍

Hexo是一个基于Node的静态博客框架。通过它我们可以快速创建自己的博客，只需要几条命令即可。发布博客，这里讲解的时部署在github上面，不仅省去服务器成本，还可以减少很多麻烦事。接下来会介绍Hexo静态博客的搭建和发布方法。

2.Hexo安装

安装Hexo之前，你需要知道准备什么。

  • 下载安装 Node.js。Node.js下载地址

    这个下载很容易，直接进入下载地址，点击下载建议版本就OK了。

    输入以下命令检查node是否安装好：

    $ node -v

    如果出现版本号就是安装好了。

  • 下载安装 npm

    因为现在新版的node.js已经自行集成npm,所以node.js安装好了npm应该也已经安装
    检查方法和node是一样的。

  • 安装Git。(安装Xcode就自带有Git了)

  • 申请GitHub账号。

3.Hexo搭建步骤

环境和工具都准备好了之后，开始搭建博客。

1.新建一个文件夹用于存放相关搭建博客的文件，例如blog。

2.在终端中执行以下命令–全局安装hexo

$ npm install -g hexo

如果以上命令出现错误，应该是需要权限，在npm前边加上sudo重新执行一次。

$ sudo install -g hexo

3.在blog文件夹下执行以下命令–建立hexo初始化相关文件

$ hexo init

之后文件里会有一些文件，目录如下：

4.修改blog目录下_config.yml文件里的一些全局配置

先修改以下几个内容


   title: 标题
   subtitle: 副标题
   description:
   author: 你的名字
   language: zh-CN

红色部分是你要修改的几个地方。

5.新建一篇博客

执行命令：

$ hexo new "第一篇博客的标题"

之后你就会在文件目录中source/_post下看到你的第一篇博客的文件，是.md格式,需要下载MarkDown软件。

直接双击打开这个md文件你就可以编辑你的第一篇博客了，记得稍微学习下md语法，挺简单的。

6.启动本地服务
命令：

$ hexo s

回车后会出现两行字母，复制其中的http://localhost:4000在浏览器网址栏中，就可以看见你的本地博客页面了。

4.hexo博客发布

上边hexo本地搭建好了，但是我们的博客是要在线上发布的，总不能只当本地的笔记使吧，所以接着就要说如何发布在github上了。

1.申请的github账号登上去，新增ssh

ssh的作用呢是为了博客上传内容时不需要每次都输入用户名和密码。

  • 在终端输入命令

    $ ssh-keygen

  • 之后三次回车会生成认证密钥对，接下来将认证所需的密钥放在文件中：

    $ vim ~/.ssh/id_rsa.pub

4.执行几个命令，博客发布完成。

设置你的用户名称和e-mail地址。这是非常重要的，因为每次Git提交都会使用该信息。它被永远的嵌入到了你的提交中：

$ git config —global user.name "你的用户名"
$ git config —global user.email  "你的email"

生成静态页面：

$ hexo g

在Hexo 3.0版本后deploy git 被分开的，所以需要安装，安装命令如下: 

$ npm install hexo-deployer-git --save

将hexo部署到github

$ hexo d

到这基本上就完成了，在浏览器网址栏输入你的用户名.github.io，就是你的博客对应的仓库名称，之后就可以看见你的博客了，别人也可以通过这个网址看到你的博客，发布成功！

以后每次博客变更发布，只需要执行hexo g和hexo d就可以了。
