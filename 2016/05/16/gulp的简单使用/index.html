gulp的简单使用

学习了gulp发现，工作中用第三方工具的好处真的很多啊，今天就简单来讲一下gulp这个工具，概念性的东西在这不多说，直接运用起来。gulp是基于任务的自动构建工具，通过插件配合解决页面压缩、JS合并、CSS文件压缩等等功能，简单又强大。

一、gulp安装

* 先安装node.js

gulp是基于Node.js的，所以你需要先安装Node.js,下载安装后，

在终端命令执行：

$ node -v

如果结果出来node版本号，就说明安装成功。

* 创建项目文件夹，执行安装gulp命令

· 新建一个文件夹，例如test,在此文件夹下执行以下命令：

$ npm install -g  gulp    //这是全局安装，去掉-g后就是局部安装。

最好全局和局部安装命令都执行一次，通常都会这样做，有人说明原因，有兴趣可以去看why do we need to install gulp globally and locally?,大体是因为版本的灵活性，不必太纠结。

如果出现以下报错，

则需要增加权限，执行以下命令：

$ sudo npm install  gulp

· 接着执行：

$ npm init   //进行初始化，建立package.json文件

· OK之后，接着执行以下命令：

$  npm install --save-dev gulp    //安装依赖模块，是项目环境

–save-dev 的作用是在安装的时候把gulp写进项目package.json文件的依赖中。

*在根目录新建gulpfile.js文件

基本上安装插件后任务都是在这个js文件中编写。

可以先来一个简单的任务：

var gulp=require('gulp');

gulp.task('default',function(){

      console.log('hello!')

   })

二、运行gulp

$ gulp

gulp一般都是配合插件使用，接下来借助两个插件为例子。

三、gulp-concat插件和gulp-jsmin插件

concat是合并插件，

jsmin是js文件压缩插件。

  • 安装gulp-concat插件

    $ npm install gulp-concat -save

  • 安装gulp-jsmin插件

    $ npm install gulp-jsmin -save
